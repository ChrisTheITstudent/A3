Andy has taken on the role of front end developer within the project. Care focus called for a UI that was easy to understand, inviting and functional. Marketing included some market research to assist in justifying the application as well as developing a launch plan and audiance capture points in marketing such as AR advertising. This role was taken on by Jennifer and also in part, Andy.

Scope and limits

The scope of the project is to develop the Care Focus application interface and databases associated with the use of the application. Within this scope, the research and learning of software used to develop the application will be required to design a prototype. While the user interface is being designed, a database that’s sharable within the team would need to be developed and a way for the application to write, edit and read from the database would need to be considered. Another factor within the scope is developing an administration interface which will enable management to observe the application data and edit as they see needed.

The final addition to the scope will include a calculation formula that the application uses to display ratios and required educators for staff. Limitations of the project would be the time it takes to learn the skills for developing the application, the current skills the team shares, software compatibility within the database and UI and access to testing methods. Another limitation of the project would be bug and the fixes required after each testing phase before the application can proceed to the next testing phase.

Testing Plans

Testing of the application initially was on an individual basis. As the team developed aspects of the application, the updated versions were made accessible to the rest of the team to explore. This resulted in some bug fixes and issues that arose regarding the accessibility of the databases both for team members and for the application itself. The second stage of testing was a group test within a meeting. This test showed that the databases required a rework before any further testing was completed.

In the future, plans for testing will include testing within a target audience which would include education professionals and parents. The target audience would ideally have a mix of both limited and moderate technical knowledge. The next testing phase is planned to be an alpha release to a early childhood education service in which the application is implemented and tested in a real world environment. After each of these phases are passed, a beta testing phase is planned in which the application is rolled out amongst several services and customer support is offered as the participant’s use and reactions are monitored. The final stage is the first release test. In this stage the application is released to the public and feedback is received. Customer support and monitoring would still be highly regarded at this stage. The final stage is planned to be evaluated and reviewed regularly.
